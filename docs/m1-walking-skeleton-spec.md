# M1 — WALKING SKELETON: SPEC & TASK PACKETS
### The first playable turn. Continuous world (D-009) at minimum viable depth: raster terrain, one settlement, a dirt path, a pop/food loop, a window, an End Turn button, one real decision. Written post-M0 per S8 §4, against kernel code-truth.

---

## 1. DECISIONS CLOSED THIS DOCUMENT

- **D-015 CLOSED — Raster resolution & physical scale.** World rasters are 1024×1024 at 4 km/px → a 4,096 km square world (Europe-class continent; fits the Scale Charter's 12–25 polities). Both numbers are data (`worldgen.json`); dev preset 256² for fast tests. Memory at 1024²: ~8 MB/layer, 6 layers ≈ 50 MB — trivial.
- **D-016 CLOSED — Catchment policy.** Catchments recompute event-driven only (network change, settlement change, terrain change) — never per-turn. Mechanism: travel-time isochrones over the **traversal lattice** (below).
- **D-022 OPENED+CLOSED — Worldgen fidelity tier.** M1 worldgen is T2-lite: plausible fields, not geological simulation. Hash-based value-noise fBm (integer SplitMix64 mixing per sample — deterministic by construction, no stream state, no trig), continental mask, sea level, latitude-and-elevation temperature, water-distance moisture, D8 flow-accumulation rivers. Explicit anti-scope: no plate tectonics, no erosion, no climate simulation — upgrade path reserved, ADR notes it.
- **D-023 OPENED+CLOSED — The Director Visual Gate.** M1 introduces packets whose acceptance cannot be fully automated in a headless container (rendering). New gate type: agent ships automated logic/view-model tests + a CI-built artifact; the director runs it and confirms specific named visual criteria. Used by T1.7, T1.8, T1.10 and the M1 exit itself.
- **D-024 PRE-AUTHORIZED — Static terrain sharing (ADR-008).** Terrain rasters are immutable after worldgen. They live in a `TerrainSet` referenced by `WorldState` but excluded from per-turn `Clone()`; their content hash is computed once at generation and folded into every `WorldHash`. Director-sanctioned refinement under the Spine's own "revisit if profiling demands" clause: cloning ~50 MB of never-changing terrain every turn buys nothing. The frozen mutable-state contract is untouched — everything that can change still double-buffers. Agent writes `docs/adr/adr-008-static-terrain.md` at T1.1.
- **Data constants (in `worldgen.json`/`sim.json`, all `TUNE`):** 1 food unit = 1 person-year of sustenance; age bands 0–15 / 15–60 / 60+; adult consumption 1.0, child 0.6, elder 0.7 food/yr; base fertility→yield conversion and demographic rates listed in T1.5.

## 2. SCOPE FENCE

**In:** worldgen fields + rivers · traversal lattice + network graph (edge type: dirt path only) · one auto-sited settlement + catchment · 3-band population + food stores + farming/consumption/demographics · labor-allocation order (farm vs path) · MonoGame+ImGui window: terrain render, overlays, HUD, End Turn · played-session order logs + replay · harness extension + new goldens · CI Windows zip.
**Out (and where it lives):** multiple settlements/cultures/classes (M2 per D-018), markets (M3), any second polity or conflict (M4), weather/climate dynamics (later), terrain mutation (later — enables ADR-008), UI beauty (perpetual; the debug UI IS the game UI).

## 3. SYSTEM NOTES (kernel-contract shapes agents must honor)

- **Worldgen is not a turn system.** It runs once, pre-turn-0, producing `TerrainSet` (immutable: elevation, water, temperature, moisture, fertility, movement-cost) + initial mutable state (network, settlement, pops, stores). Fully deterministic from the world seed; two generations of the same seed must be byte-identical (worldgen twin-test).
- **Traversal lattice (implementation detail, never design-visible).** Stride-4 sampling of the movement-cost field → 256² lattice graph; network edges overlay as fast lanes. All pathfinding and isochrones run here. D-012's precedent applies: an under-the-hood grid is legal; a player-visible or sim-unit-of-place grid is not. **Determinism mandate:** priority queues break ties by node id — total ordering, no insertion-order reliance.
- **Turn pipeline (M1 production preset):** Farming → Consumption → Demographics → PathBuild. The T0.x toy systems retire to a `pipeline.toy.json` preset the kernel-invariant harness tests keep using; production harness runs the M1 preset. Pipeline presets are data (T0.5 loader).
- **Orders get real.** `LaborAllocationOrder {farmPct}` — the M0 synthetic order retires. UI and CLI both emit it through the T0.7 order-log pipe; the sim never reads the UI.
- **UI is a view + order source, single-threaded.** End Turn runs the executor synchronously (turns are sub-millisecond); the window renders the latest completed state. No UI state ever enters `WorldState`.

## 4. TASK PACKETS (one session each; adversarial-mandatory: T1.1, T1.3, T1.5, T1.9)

**T1.1 — Worldgen fields + ADR-008.** `TerrainSet`, generation pipeline (continental mask → elevation fBm → sea level → temperature → moisture → fertility v0 → movement-cost), `sim worldgen --seed N --stats` CLI. *Accept:* worldgen twin-test byte-identical; land fraction within TUNE bounds across 10 seeds; 1024² generation < 5 s; stats print min/max/mean per field; ADR-008 written; WorldHash folds the terrain hash (proven: different seed → different hash, same seed → stable).
**T1.2 — Hydrology.** D8 flow accumulation over elevation (deterministic sink handling documented), top-K river polylines by discharge, river-adjacency fertility boost, rivers into `TerrainSet`. *Accept:* rivers flow monotonically downhill to sea/sink; twin-test still byte-identical; fertility field visibly river-correlated in stats.
**T1.3 — Lattice + network.** Movement-cost lattice, `NetworkNodes`/`NetworkEdges` tables (dirt-path type), A*/Dijkstra with the tie-break mandate, travel-time + isochrone queries, D-016 event-driven catchment recompute hooks. *Accept:* path determinism property test (FsCheck: random endpoint pairs, twin runs, identical paths); triangle-inequality sanity; isochrone from a point is deterministic and contiguous; documented big-O and measured ms at 1024².
**T1.4 — Settlement + catchment.** `Settlements` table, deterministic auto-siting (argmax fertility × water access, id tie-break), catchment isochrone (travel-time budget TUNE), effective-farmland aggregation sampling fertility within catchment. *Accept:* same seed → same site; site is on land, near water across 10 seeds; catchment area responds to a hand-added path edge in a test (the dirt path provably matters).
**T1.5 — Population + food.** `PopBands` (3 conserved long stocks), `FoodStore`, FarmingSystem (yield = effective farmland × farm-labor share × conversion × dt), ConsumptionSystem (band-weighted, dt-scaled, ClampToAvailable), DemographicsSystem (births from adults, aging flows between bands, base + starvation mortality; all Ledger flows with named reasons; remainder accumulators in rows per T0.5 precedent). Conservation quantities registered: Population, Food. *Accept:* fed world grows; unfed world starves to a floor, never negative, never NaN; Malthus-lite: overshoot → correction oscillation visible in a 200-turn run; per-turn audit exact; dt-halving property holds for demographic flows.
**T1.6 — Labor order + PathBuild.** `LaborAllocationOrder` end-to-end (log → OrderBatch → systems); PathBuildSystem: banked path-labor extends the network edge toward best unreached fertility (deterministic target choice, id tie-break), triggering D-016 recompute. *Accept:* allocation shifts farming yield and path growth measurably and dt-correctly; replay of an ordered headless run is hash-identical; catchment grows when the path extends.
**T1.7 — Sim.Ui: window + terrain render. [Director Visual Gate]** MonoGame DesktopGL + ImGui.NET; camera pan/zoom; smooth-shaded terrain (hypsometric tint + water + rivers; explicitly no visible tiles). Agent ships view-model tests + CI artifact; *Director confirms:* continuous-looking continent, rivers visible, 60 fps feel, zoom smooth. Record any MonoGame TFM pin as an ADR if needed.
**T1.8 — Sim.Ui: overlays + HUD + End Turn. [Director Visual Gate]** Settlement marker + footprint dot, path polyline, catchment overlay toggle; ImGui HUD: population, food store, last-harvest; labor slider issuing real orders; End Turn button; session order-log autosave to a run folder. *Director confirms:* a played 20-turn session where moving the slider visibly changes outcomes.
**T1.9 — Replay + harness extension + goldens.** Played-session replay (`sim replay` consumes a UI-produced log, hash-identical); twin/xproc/conservation harness runs the M1 production preset headless; new golden hash pinned (deliberate update, comment per T0.7 rule); toy-preset kernel tests kept. *Accept:* all harness suites green on M1 systems; a real UI session's log replays hash-identical; CI green.
**T1.10 — CI Windows artifact. [Director Visual Gate]** `dotnet publish -r win-x64 --self-contained` of Sim.Ui, zipped, uploaded as an Actions artifact on every main merge; README "Download & Play". *Director confirms:* downloads zip on the Victus, unzips, plays — no toolchain touched. This closes the promise that playing never requires the SDK.
